MongoDB setup on OSX

Install Homebrew


/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

Paste that at a Terminal prompt.

Homebrew installs the stuff you need that Apple didnâ€™t. To learn more visit https://brew.sh/

Install MongoDB


brew install mongo

Paste that at a Terminal prompt.

By default MongoDB requires you to have a directory on your hard drive where it can stash all your data. Therefore, we have to manually create the directory. To do so paste the code below at a Terminal prompt. May be prompted for your password to execute.


sudo mkdir -p /data/db

If permission is denied it means you have to take ownership of that directory (computer acknowledges that the path exists but questions whether you have the credentials to access it).


whoami

Paste that at a Terminal prompt. This will tell you the username of the currently logged in user. When you know you username you can run a command to take ownership of the /data/db directory.


sudo chown -Rv [insert returned username here without brackets] /data/db

-Rv simply prints out a list of all the directories updated

Start MongoDB

It is important to note that to work with MongoDB there are two important components. First is the Mongo Server mongod and the Mongo shell mongo.

You first need to startup the mongo database (via mongod) before you can interact with it (via mongo shell mongo)

On a Mac/Linux operating system there are two ways you can start Mongo

  1. mongod
  2. brew services start mongo

First approach starts up an instance of the MongoDB server. So each time you want to interact with the database you would have to make sure the server is on. Second approach tasks your computer (via services) to turn start the MongoDB server whenever you turn on your computer. In other words, the MongoDB server is always on in the background.

Connect to MongoDB.

When the database is turned on run mongo to open up the MongoDB shell
